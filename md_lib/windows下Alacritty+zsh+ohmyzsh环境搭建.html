Alacritty

首先 我是重度Alacritty+zsh用户(macos下用iterm)

所以 在windwos上自然也是要用

并且支持使用%APPDATA%\alacritty\alacritty.yml来自定义你的Alacritty

WSL

适用于 Linux 的 Windows 子系统 (WSL) 可让开发人员直接在 Windows 上按原样运行 GNU/Linux 环境（包括大多数命令行工具、实用工具和应用程序），且不会产生传统虚拟机或双启动设置开销。

现在微软有两款WSL分别是WSL1和WSL2

比较 WSL 1 和 WSL 2

功能 WSL 1 WSL 2
Windows 和 Linux 之间的集成 ✅ ✅
启动时间短 ✅ ✅
与传统虚拟机相比，占用的资源量少 ✅ ✅
可以与当前版本的 VMware 和 VirtualBox 一起运行 ✅ ✅
托管 VM ❌ ✅
完整的 Linux 内核 ❌ ✅
完全的系统调用兼容性 ❌ ✅
跨 OS 文件系统的性能 ✅ ❌

为什么我用WSL1而不用WSL2？

因为我认为WSL2的优势可以总结为

  • 可以调用GPU从而使用CUDA
    • 但是性能损耗大概在18%的样子而且配置起来很奇怪
  • 简单概括就是更好的性能
    • 我认为这更好的性能 不大于 OS文件系统调用的便捷性

适合自己的才是最好的
