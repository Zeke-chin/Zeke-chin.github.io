Git

什么Git

Git是一个分布式版本控制软件

简单来说 就是用来进行时光倒流的后悔药

除此之外 还可以当作行走的配置文件

而不得不提的是Git之父Linus Torvalds大神

是的没错 就是那个’linux之父’ 仅用了10天时间！就把git开发出来了

Git的安装和配置

  • 在安装之我们要先确认我们有没有Git

    • 在linux系统下大多数情况下会安装在/usr/bin/git目录下
    • 在mac系统下比较主流的有两种Git
      • 来自homebrew的Git
        • 安装在/opt/homebrew/bin/git目录下
      • 来自Xcode的Git
        • 安装在/usr/bin/git
        • 用git --version查看的话回发现是git version 1.9.3 (Apple Git-26)
    • 值得一提的是在mac中我们一般使用brew版的Git原因是Xcode版的Git不大好升级和管理，所以我们需要进行额外的配置

    Git安装

    在linux系统下

    • 很简单一行代码(ubuntu为例其他发行版本差不多)

      sudo apt install git

    在mac系统下

    • 使用homebrew

      brew install git

git配置

Git查看配置命令

Git的config有三个等级,系统级、用户级、仓库级

system->global->local

使用顺序是 底层覆盖顶层配置

  • git config -l 全部配置

  • git config –system –list 系统配置

下面进行必须的Git配置

  • 设置(系统级)签名
    • git config –global user.name “yourusername” #名称
    • git config –global user.email “your_email@example.com“ #邮箱
    • 这里的配置只是签名 并没有什么实质意义只是证明这个提交是你干的 是你的地址
      • (如果做错的 方便别人根据你的地址来找你[doge])
  • 以github为例绑定SSH密钥，其他

    • 在终端输入 ls -al ~/.ssh查看是否拥有公共 SSH 密钥

      • 如果有点击跳转到绑定到Github

      • 如果粘贴下面的文本，替换为您的 GitHub 电子邮件地址。

        • ssh-keygen -t ed25519 -C "your_email@example.com"
        • 接着在后台启动ssh_agent
        • 将你的 SSH 私钥添加到 ssh-agent
        • 将 SSH 密钥添加到您在 GitHub 上的帐户

        • 将 SSH 公钥复制到剪贴板

Git命令

  • 创建仓库

    • 本地创建->git init
    • clone远程->git clone [url ]

  • Git 文件状态

    • Untracked ——未跟踪
      • 野生文件 可添加到Stage(暂存区)
        • 被标记git add [file]==>Untracked–>Staged
    • Unmodify ——stage内文件未修改
      • 标记文件 和Untracked一样
        • 被改动cont+s==>Untracked–>Modified
        • 被移除git rm [file]==>Untracked–>Untracked
    • Modified ——stage内文件已修改
      • 标记文件 和Untracked不
        • 被确实git add .==>Modified–>Staged
        • 被丢弃gie checkout [file]==>Modified–>Unmodify
    • Staged ——stage内文件
      • 暂存文件 等待被提交同步或取消暂存
        • 被提交git commit -m [message] ==>Staged–>Unmodify
        • 被取消git reset HEAD [file]==>Staged–>Modified
